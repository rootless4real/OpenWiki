Congratulations!

So far so good.

The installation program was able to create a new virtual directory in your default website. It has been given the name OpenWiki. Also, a new System DSN has been created which you can see in your ODBC Data Source Administrator. It has been given the name OpenWikiDist.

If your default website is running then you should be able to work with your new OpenWiki site right away. Try it out at http://localhost/OpenWiki.

If your default website is running, yet OpenWiki doesn't work correctly then you probably do not have all the required components installed yet. Before OpenWiki can run you must have the following components installed:

  • MDAC 2.5 or higher.
  • VBScript v5.5 or higher.
  • MSXML Parser v3.0 SP 2 or higher.
  • A Web Server (i.e. PWS or IIS).
  • A database like MS Access, SQL-Server, Oracle or MySQL.

Usually on a standard installed Windows OS all these components are already present. That is, for the database you'd at least have (the drivers of) MS Access.

Furthermore, if you want to allow file uploads to your wiki, then the following component must be installed:

  • ABCUpload v4.1 or higher.

Change the configuration of your OpenWiki site by modifying the values in file:
<install-dir>/owbase/ow/owconfig_default.asp.

More help can be found in the README.txt and INSTALL.txt files.
